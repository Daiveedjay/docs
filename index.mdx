Keep AI-assisted code up to your standards. Iris Code catches secrets, duplicate code, security smells, and maintainability regressions before they reach human review. Source analysis runs locally, and the same deterministic rules run in VS Code, the `iris` CLI, hooks, and CI.

<Note>
  Iris Code does not try to identify AI-generated code. It independently checks the result with explicit rules, whether a person, an AI assistant, or both wrote it. Your source is not uploaded to an AI model or analysis backend.
</Note>

## What Iris Code Does

Iris Code continuously monitors the files you work on and the projects you manage, giving you four interconnected layers of insight:

**Health scores** - every supported file receives a 0–100 composite score on every save, penalising long functions, deep nesting, debug prints, magic numbers, and secrets. The score updates instantly; no manual trigger required.

**Code smells** - Iris Code detects TODOs, unused code, magic numbers, debug print statements, and long parameter lists across all supported languages. Every finding is clickable and jumps you directly to the relevant line.

**Secrets detection** - hardcoded credentials are flagged with a masked preview in the sidebar before they can ship. Secrets detection is free on all plans and works without any configuration.

**Enforcement** - Gate Preview lets you preview your enforcement posture against five preset thresholds before installing a single hook. When you're ready, a git pre-push hook or build hook blocks files that fall below your chosen minimum health score. The same threshold can also be enforced in CI via `iris gate`.

## How Iris Code works

<Steps>
  <Step title="Install the extension">
    Install [Iris Code](https://marketplace.visualstudio.com/items?itemName=davidjaja.iris-code) from the VS Code Marketplace - it's free. The Iris Code icon appears in the Activity Bar immediately.
  </Step>
  <Step title="Open any file">
    Open a JavaScript, TypeScript, Go, or Python file. Iris Code scores it instantly in the sidebar - health score, complexity, functions, code smells, and hardcoded secrets with no configuration required.
  </Step>
  <Step title="Preview your enforcement posture">
    Scroll to the bottom of the File tab and click **Preview Gate**. See how your codebase scores against all five built-in presets before enabling a single hook.
  </Step>
  <Step title="Block weak code automatically">
    Install a git pre-push hook or build hook from the command palette - or run `iris gate` in CI. Pushes and builds that fall below your threshold are blocked before they ship.
  </Step>
</Steps>

Iris Code performs source analysis in-process on your machine. Your source code never reaches an analysis backend, and no AI model processes your files. Network requests are limited to account and licensing operations, dependency and advisory lookups, and product-use events. These requests never include source code, file contents, or project structure; you can disable optional product-use events in your account settings.

This means:

- **Instant feedback** - analysis runs in milliseconds on every save, with no round-trip latency
- **Offline capable** - file analysis and CLI commands work without an internet connection
- **Private by design** - proprietary code, secrets, and unreleased features stay on your machine

## Supported Languages

Iris Code provides full analysis for four languages:

| Language | File extensions | Notes |
|---|---|---|
| **TypeScript** | `.ts`, `.tsx` | Includes `any` usage, `@ts-ignore`, non-null assertions, and missing return types |
| **JavaScript** | `.js`, `.jsx`, `.mjs`, `.cjs` | Full smell and complexity analysis; unused import detection included |
| **Go** | `.go` | Method and function detection, `go.mod` unused package audit, `vendor/` skipped automatically |
| **Python** | `.py` | `def` declarations, `requirements.txt` / `pyproject.toml` package audit, `.venv` skipped automatically |
