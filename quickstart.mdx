This guide walks you through everything you need to get Iris running: installing the extension, reading your first file score, scanning your workspace, and previewing your enforcement posture against five built-in thresholds. By the end you'll have a clear picture of where your codebase stands ŌĆö and what it would take to enforce a gate on it.

<Steps>
  <Step title="Install Iris from the VS Code Marketplace">
    Open the Extensions panel in VS Code with `Ctrl+Shift+X` (or `Cmd+Shift+X` on macOS), search for **Iris**, and click **Install**. The Iris icon appears in the Activity Bar as soon as the install completes ŌĆö click it to open the sidebar.

    <Note>
      Iris installs in any VS Code-compatible editor. If you use **Cursor**, **Windsurf**, or **VSCodium**, the Marketplace install works the same way ŌĆö or you can [install the `.vsix` manually](/installation).
    </Note>
  </Step>

  <Step title="Open a supported file and read the File tab">
    Open any `.js`, `.jsx`, `.ts`, `.tsx`, `.go`, or `.py` file. Iris scores it automatically ŌĆö no command needed.

    The **File tab** in the Iris sidebar shows:

    - **Enforcement snapshot** ŌĆö a top-of-file readiness summary with your score, blockers, hook posture, and trend direction
    - **Health score** ŌĆö a 0ŌĆō100 composite score recomputed on every save
    - **Blockers and warnings** ŌĆö findings that penalise your score, each clickable to jump to the source line
    - **Function inventory** ŌĆö every detected function with its line number, length, and complexity
    - **Code smells** ŌĆö TODOs, magic numbers, debug prints, unused variables, and long parameter lists grouped in one section

    Save the file and watch the score update in real time.
  </Step>

  <Step title="Run a workspace scan">
    Open the command palette (`Ctrl+Shift+P` / `Cmd+Shift+P`) and run **Iris: Analyse Workspace**.

    Iris crawls every supported file in the project and populates:

    - **Workspace Readiness** ŌĆö an aggregate health score with blocker and warning counts across the whole project
    - **Hotspot rankings** ŌĆö the largest and most complex files ranked first, so you know where to focus
    - **Unused packages** ŌĆö dependencies declared in `package.json`, `go.mod`, `requirements.txt`, or `pyproject.toml` that are never imported
    - **Issues tab** ŌĆö all blocking issues and warning-level findings aggregated in one filterable list
    - **TODOs tab** ŌĆö every TODO, FIXME, and HACK comment across your project

    Results are cached ŌĆö navigate between tabs and come back without re-scanning.
  </Step>

  <Step title="Preview your enforcement posture">
    Scroll to the bottom of the **File tab** and click **Preview Gate**. Iris runs a background workspace scan and renders your posture against all five built-in presets in a read-only view:

    | Preset | Min score | Best for |
    |---|---|---|
    | **Legacy** | 60 | Large codebases with accumulated debt |
    | **Balanced** | 70 | A typical TypeScript project ŌĆö best starting point |
    | **TypeScript** | 76 | Raises the bar on type safety |
    | **Strict** | 82 | Greenfield code or shared libraries |
    | **Security** | 85 | Prioritises secrets and safety |

    Each card shows exactly how many files would be blocked at that threshold. When you find the right preset, click **Apply** to write `presetId` to your `.irisconfig.json` ŌĆö no config editing required.

    <Note>
      Gate Preview is free for all users and always read-only. No hooks are installed and no files are blocked until you explicitly set up enforcement. Pro users can drill into the list of failing files and their scores for each preset.
    </Note>
  </Step>

  <Step title="Install the CLI (optional)">
    To scan files, gate CI builds, and audit dependencies from the terminal without opening VS Code, install the Iris CLI:

    ```bash
    npm install -g @iris-code/cli
    ```

    Verify the install:

    ```bash
    iris --version
    ```

    The CLI shares the same thresholds as the extension. Run `iris check <file>` for a single-file scan, or `iris gate` in a CI pipeline to enforce your health threshold before merging.
  </Step>
</Steps>
